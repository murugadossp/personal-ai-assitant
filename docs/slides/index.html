Brief about the Idea

Natural language over your real Google Workspace

The project is a personal AI assistant that lets you talk to Calendar, Gmail, and Drive in plain English—without juggling separate apps or brittle custom scripts.

A coordinator agent handles routing and light conversation, then hands workspace tasks to a single workspace sub-agent. That agent calls a dedicated Google Workspace MCP server over Streamable HTTP, keeping OAuth, tokens, and API details out of the agent’s core logic.

Built for Google GenAI APAC Hackathon 2026: demonstrate ADK multi-agent patterns, MCP as a clean tool boundary, and production-minded touches (config-driven Gemini models, deployable MCP and assistant services).

Architecture

Coordinator, one workspace agent, MCP outside

The ADK app root is the coordinator: it classifies intent, handles chit-chat, and delegates with transfer_to_agent("workspace_agent"). There are no separate Calendar/Gmail/Drive agents—one workspace specialist carries the full MCP toolset.

User → Coordinator → Workspace agent → Workspace MCP → Google APIs

The Workspace MCP server exposes tools (calendar_*, gmail_*, drive_*), runs OAuth and token storage, and talks to Google REST APIs. The assistant stays focused on reasoning and tool orchestration—not low-level auth plumbing.

  • Transport: MCP Streamable HTTP between assistant and server.
  • Deploy: assistant and MCP as separate services (e.g. two Cloud Run services) with mcp_settings.json pointing at the live MCP URL.
