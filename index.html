Tech Stack

🧠 Gemini (config-driven)

Models are defined in personal-ai-assistant/config.yaml — separate blocks for coordinator and workspace (defaults use the same chain). ADK: coordinator + workspace sub-agent; MCP tools only on the workspace agent. Multiple IDs use FallbackGeminiLlm.

  • Primary: first model_candidates entry serves every turn until it errors.
  • Fallbacks: on quota / rate-limit style errors (e.g. 429), the next entry is tried automatically.
  • Customize: reorder, add, or trim the list; one entry = no fallback wrapper.

See personal-ai-assistant/docs/model-fallback.md · e.g. gemini-3.1-flash-lite-preview → gemini-3-flash-preview → gemini-2.5-flash

🔌 Protocol

Model Context Protocol (MCP) via Streamable HTTP. Official Python SDK (mcp v1.27). FastMCP decorators.

🔐 Security

OAuth 2.0 per-user authorization. Firestore token storage in production. Cloud Run IAM access control.

☁️ Deployment

Google Cloud Run containers. Firestore for token persistence. Auto-scaling and HTTPS built-in.

🐍 Runtime

Python 3.12+ with isolated venvs. UV package manager. Starlette ASGI server.

🌐 APIs

Google Calendar API v3. Gmail API v1. Drive API v3. All via REST + google-api-python-client.

Quick Start
1

Start MCP Server

cd mcp-servers/google_workspace source .venv/bin/activate MCP_TRANSPORT=streamable-http python server.py
2

Start AI Assistant

cd personal-ai-assistant source .venv/bin/activate ./scripts/adk_web.sh # or: python -m app.main (FastAPI /api/chat; set ENABLE_ADK_WEB_UI=true for UI)
3

Authorize Google (first time)

Open your MCP base URL, e.g. http://localhost:8080/auth?user_id=… locally, or https://<your-mcp-run.app>/auth?… in production.

4

Start chatting!

Ask: "What's on my calendar?" or "Show my unread emails"
